## Reportar una Vulnerabilidad

La seguridad de KeyGo Server es nuestra prioridad máxima. Si descubres una vulnerabilidad de seguridad, por favor sigue estos pasos:

### 1. **NO** abras un issue público

Las vulnerabilidades de seguridad deben ser reportadas de forma privada.

### 2. Envía un reporte privado

Contacta al mantenedor directamente:
- **Email**: Crea un issue privado en GitHub o contacta via GitHub
- **Asunto**: [SECURITY] Descripción breve

### 3. Incluye en tu reporte

- Descripción detallada de la vulnerabilidad
- Pasos para reproducir el problema
- Versión afectada
- Posible impacto
- Sugerencias de solución (si las tienes)

### 4. Tiempo de respuesta

- **Confirmación inicial**: 48 horas
- **Actualización de estado**: 7 días
- **Resolución estimada**: 30 días (según severidad)

## Proceso de Divulgación

1. Recibes confirmación de tu reporte
2. Investigamos y validamos la vulnerabilidad
3. Desarrollamos y probamos el fix
4. Publicamos la versión corregida
5. Publicamos un advisory de seguridad
6. Te damos crédito (si lo deseas)

## Prácticas de Seguridad

Este proyecto sigue estas prácticas:

- ✅ Dependencias actualizadas regularmente
- ✅ Análisis de vulnerabilidades con herramientas automatizadas
- ✅ Code reviews obligatorios
- ✅ Tests de seguridad en CI/CD
- ✅ Principio de mínimo privilegio
- ✅ Cifrado de datos sensibles

## Actualizaciones de Seguridad

Las actualizaciones de seguridad se publican:
- Como versiones patch (x.x.X)
- Con notas de release detalladas
- En la sección [Security Advisories](https://github.com/cmartinezs/keygo-server/security/advisories)

## Agradecimientos

Agradecemos a los investigadores de seguridad que reportan vulnerabilidades de forma responsable.

---

**Nota**: Este proyecto maneja autenticación y datos de usuarios. La seguridad es crítica.
