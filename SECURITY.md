### Reporting a Vulnerability

KeyGo Server security is our top priority. If you discover a security vulnerability, please follow these steps:

#### 1. **DO NOT** open a public issue

Security vulnerabilities must be reported privately.

#### 2. Send a Private Report

Contact the maintainer directly:
- **Email**: Create a private issue on GitHub or contact via GitHub
- **Subject**: [SECURITY] Brief description

#### 3. Include in Your Report

- Detailed description of the vulnerability
- Steps to reproduce the problem
- Affected version
- Potential impact
- Solution suggestions (if you have any)

#### 4. Response Time

- **Initial confirmation**: 48 hours
- **Status update**: 7 days
- **Estimated resolution**: 30 days (depending on severity)

### Disclosure Process

1. You receive confirmation of your report
2. We investigate and validate the vulnerability
3. We develop and test the fix
4. We publish the fixed version
5. We publish a security advisory
6. We give you credit (if you wish)

### Security Practices

This project follows these practices:

- ✅ Regularly updated dependencies
- ✅ Vulnerability analysis with automated tools
- ✅ Mandatory code reviews
- ✅ Security testing in CI/CD
- ✅ Principle of least privilege
- ✅ Encryption of sensitive data

### Security Updates

Security updates are published:
- As patch versions (x.x.X)
- With detailed release notes
- In the [Security Advisories](https://github.com/cmartinezs/keygo-server/security/advisories) section

### Acknowledgments

We thank security researchers who report vulnerabilities responsibly.

---

**Note**: This project handles authentication and user data. Security is critical.
